Comment 3:
  Author: riklaunim
  Comment: * You should avoid using globals in your code.
* Write tests for your code and write clean and easy to maintain/test code
* Follow pep8 and linter recommendations for code style
* tkinter is not the best thing for UI nowadays, especially if you are looking for a commercial career
* Don't be afraid to commit - write clear and understandable commit messages, if needed add commit comments to explain reasoning behind given commit. Don't commit big blobs of multiple changes, don't publish 1-commit projects that then get abandoned.

A shop will have a database of good available and POS system will scan product barcodes and match their serial numbers with records in the database and then get the price, tax rate, update stock and whatever else.

Usually products come with their own barcodes and if needed there are barcode sticker printers so either way adding a new product or updating stock will also work based on scanning the barcode and updating the data.

Desktop GUI can be a thing but for smaller systems it can be something embedded or for bigger systems - web apps (which are easier to prototype). In either way it will very likely not run on local data but access some storage over the network - either local or external.

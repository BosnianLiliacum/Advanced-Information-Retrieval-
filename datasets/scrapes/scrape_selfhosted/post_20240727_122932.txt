  Author: cyt0kinetic
  Comment: I am still working on my "offsite", I may even just do cold storage in a safe location, the personal data only changes occasionally in a way where there aren't local copies on other devices. Though I have an independent backup on the network, with a drive from a different manufacturer.

So I have a Raspberry Pi with a backup shell script using rsync. It runs nightly. It creates a read only SMB connection with the main data drive partitions, mounts the corresponding backup partition. Runs an incremental backup, updates the primary backup with the oldest incremental backup, deleted the oldest backup, then umounts the drive. I wrote it as a function so for our 4 partitions I just need to call it for each versus rescript the whole thing. Lots of how tos out there.

This folder is monitored via synching (https://syncthing.net), and every change in that folder is instantly replicated (one side) to my cloud machine (Oracle Always Free Tier)/   
Additionally, just in case something happens with my cloud machine, the same folder is uploaded to Google Drive using "Kopia" (https://kopia.io/)

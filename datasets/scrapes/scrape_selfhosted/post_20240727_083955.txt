Comment 2:
  Author: jusepal
  Comment: Start learning linux. Not that selfhosting on windows aren't possible but for ease of finding tools, docs and general help, linux is the goto.

Pick a virtual machine software, install it on your windows host, create a vm and install a linux distro on it. For the vm software something like virtualbox or vmware could work.

For the linux distro itself, you could either start with gui distro or their cli no gui counterparts but I'd go straight cli since thats the goto for linux low level stuff. Theres too little to learn on gui linux since its basically 90% similar to windows and you'll eventually open terminal too so just go cold turkey cli. For the linux distro itself, again, for ease of finding documentation pick debian or debian based distro. Personally I'd start with ubuntu server.

Basically you install the linux distro on the vm, clone the vm for backup, play around and fck things up, restore again using the backup'ed vm image, repeat.

Learn about linux permission, networking in the vm. Howto manage permissions, howto manage users, howto manage firewall, howto manual/auto update system, that kind of stuff. Learn commands for either vi or nano too.

Then proceed to learn hosting basic stuff; bind9, nginx etc. Then maybe proceed to learn container; docker, podman, kubernetes etc.

Don't be scared, create vm image backup before doing anything substantial. If you bork things up, restore and continue.

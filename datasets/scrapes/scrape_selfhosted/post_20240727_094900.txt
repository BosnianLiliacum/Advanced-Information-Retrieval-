Can you clarify your situation. Is the current working instance installed on bare metal? Or is the current working instance utilizing docker?

If the current working instance is utilizing docker on the old machine, you should be able to copy all volumes that are associated with the docker compose/ docker image and migrate that to the new server (ensure it is in the exact location that is defined in the docker compose/ docker command and has the exact same permissions)

This is why people utilize docker, for easy migration 

Hope that helps
